# Лабораторная работа 1. Знакомство с IaaS, PaaS, SaaS сервисами в облаке на примере Amazon Web Services (AWS). Создание сервисной модели.
## Теория
Разобрались с основными моделями облачных вычислений. \
Модели расположены в порядке возрастания уровня абстракции и, наоборот, снижения уровня контроля пользователя:
* Iaas (infrostructure as a service) - сервис предоставляет инфаструктуру, по сути "виртуальное железо"
* Paas (Platform as a service) - сервис предоставляет платформу для разарботки/хранения/...
* Saas (Software as a service) - сервис предоставляет возможность обращаться к готовому ПО, запускаемому на сервере
## Ход работы
Разобрались с примером, изучали документацию, гуглили.
### Amazon Elastic Compute Cloud
Сервис для облачных вычислений, предлагает глубоко кастомизируемые виртуальные машины; Iaas. Вот основные пункты из данны билинга: 
* Скидочные прораммы, при условии гарантии использования сервиса на определенный срок (оплата сразу или рассрочка)
* Dedicated instance и dedicated host, предоставляется instance на сервере или собственно сервер.
* Spot-instances, использование "неактуальных" серверов по скидке
* Возмоность ускорять вычисления с помощью использования GPU
* CPU-кредиты, система получения "моенток" при низком использовании ресурсов и возможность тратить их для ускорения вычислений когда это потребуется.
Российские аналоги - Yandex Cloud, Selectel, VK cloud, ...
### Amazon Cloud Front
Сервис ускорения доставки контента, Saas. Основные направления:
* Запросы разного типа в разном количестве, в зависимости от tier-а
* Подписки
* Кастомные SSL-сертификаты
Российский аналог - Yandex CDN
### Amazon Kendra
Сервис ии-поиска, Paas. Основные направления:
* Чтение и поиск по документам (на естественном языке)
* Подключение к БД и репозиториям
Российский аналог - Yandex Search.
### Amazon Lex
Платформа для создания ИИ-чатботов, Paas. Основные направления:
* Собственно создание ботов
Российский аналог - Яндекс диалоги.
### Amazon Route 53
DNS сервис, Paas. Основные направления:
* Проверка возможностей/состояния сервера (health checks)
* Запросы DNS, GEO-DNS (разный IP в зависимости от положения пользователя), LBR (распределяет трафик по нескольким серверам)
* Resolver (DNS-сервис внутри частой сети)
Российский аналог Reg.ru DNS
