# Лабораторная работа 2. Сравнение сервисов Amazon Web Services и Microsoft Azure. Создание единой кросс-провайдерной сервисной модели.

## Структура сервисов Azure
### Cloud Services (Аналог в AWS: Compute)
Назначение: Основная категория для вычислительных, аналитических и платформенных сервисов.

Семейства сервисов:
1) **Application Services** - сервисы для развертывания приложений

 - **Azure Container Instance** (запуск контейнеров без управления инфраструктурой)

Классификация: **PaaS**, т.к. пользователь управляет только контейнерами и приложениями, не ОС/сетью

Тип оплаты: **Core Duration, vCPU Duration** - оплата за время работы процессорных ядер

**Memory Duration** - оплата за время использования памяти

**Executions** - оплата за каждый запуск контейнера

2) **Developer Tools** - управляемые сервисы для разработчиков: реестры, репозитории, инструменты CI/CD.

 - **Azure Container Registry** - хранение и управление образами контейнеров

Классификация: **PaaS**, т.к. реестр управляемый, инфраструктура скрыта

Тип оплаты: **Registry Unit** (Small/Basic/Standard/Premium) - фиксированная плата за зарезервированный уровень производительности (емкость, пропускная способность)

3) **Analytics** - сервисы для сбора, обработки, анализа или визуализации данных.

 - **Azure Data Lake Analytics** - анализ больших данных в озере данных с использованием языков U-SQL, .NET

Классификация: **PaaS/SaaS**, т.к. предоставляется настраиваемый движок для анализа данных в озере

Тип оплаты: **Pay-as-you-go Analytics Units** - consumption-based модель, плата за единицы аналитики (AU), потребленные заданием

 - **Azure Data Lake Storage** - масштабируемое хранилище больших данных

Классификация: **PaaS/SaaS**, т.к. предоставляется платформа для хранения и обработки данных, но можно рассмотреть и как готовое хранилище

Тип оплаты: **Data Storage, Transactions** - оплата за объем хранимых данных и/или за транзакции

 - **Azure Data Lake** - экосистема сервисов платформы Asure Data Lake (Data Transfer, Management и т.д.)

Классификация: **PaaS**, т.к. управляемая платформа для работы с большими данными

Тип оплаты: **Geo Replicated, Geo Redundant, Data Management, Data Services** - плата за дополнительные сервисы обработки данных (репликацию, избыточность и т.д.)

 - **Dynamics 365 Customer Insights** - AI-платформа для анализа данных клиентов и построения единого профиля клиента 360°

Классификация: **SaaS**, т.к. готовое CRM-решение с искусственным интеллектом для аналитики клиентских данных, не требующее управления инфраструктурой или разработки приложений

Тип оплаты: **Daily Unit** - оплата за ежедневное использование сервиса, зависящая от объёма обрабатываемых данных и количества анализируемых клиентских профилей


4) **Artificial Intelligence** - платформы и инструменты для машинного обучения и AI

 - **Azure Machine Learning** - платформа для создания, обучения и развертывания ML-моделей

Классификация: **PaaS**, т.к. предоставляются инструменты и управляемая среда для работы с ML, а не готовая модель

 - **Azure AI Search** - служба поиска с искусственным интеллектом для создания мощных поисковых решений по корпоративным данным

Классификация: **PaaS**, т.к. предоставляет платформу для создания собственных поисковых решений (настройка индексов, схем, релевантности), а не готовый поисковик

Тип оплаты: **Bandwidth** - оплата за исходящий трафик (экспорт обученной модели, передача данных кленту)

Классификация некоторых сервисов зависит от использования, например:

 - **Azure Network Watcher** - мониторинг, диагностика и метрики для сетевых ресурсов

Классификация: **PaaS**, т.к. предоставляется набор инструментов и API для анализа сети, требующий конфигурации

Тип оплаты: **Diagnostic API Tool** - оплата за диагностику (Cloud Services, Analytics)
**Network Watcher** - оплата за операцию healthcheck (Networking)

Также сервис **Azure Operational Insights** из категории Cloud Services, Analytics интегрирован Health Check функцией с Network Watcher
