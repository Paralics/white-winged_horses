# Лабораторная работа 2. Сравнение сервисов Amazon Web Services и Microsoft Azure. Создание единой кросс-провайдерной сервисной модели.

## Структура сервисов Azure
### Cloud Services (Аналог в AWS: Compute)
Назначение: Основная категория для вычислительных, аналитических и платформенных сервисов.

Семейства сервисов:
1) **Application Services** - сервисы для развертывания приложений

 - **Azure Container Instance** (запуск контейнеров без управления инфраструктурой)

Классификация: **PaaS**, т.к. пользователь управляет только контейнерами и приложениями, не ОС/сетью

Тип оплаты: **Core Duration, vCPU Duration** - оплата за время работы процессорных ядер

**Memory Duration** - оплата за время использования памяти

**Executions** - оплата за каждый запуск контейнера

Аналог: AWS Fargate

2) **Developer Tools** - управляемые сервисы для разработчиков: реестры, репозитории, инструменты CI/CD.

 - **Azure Container Registry** - хранение и управление образами контейнеров

Классификация: **PaaS**, т.к. реестр управляемый, инфраструктура скрыта

Тип оплаты: **Registry Unit** (Small/Basic/Standard/Premium) - фиксированная плата за зарезервированный уровень производительности (емкость, пропускная способность)

Аналог: Amazon Elastic Container Registry

3) **Analytics** - сервисы для сбора, обработки, анализа или визуализации данных.

 - **Azure Data Lake Analytics** - анализ больших данных в озере данных с использованием языков U-SQL, .NET

Классификация: **PaaS/SaaS**, т.к. предоставляется настраиваемый движок для анализа данных в озере

Тип оплаты: **Pay-as-you-go Analytics Units** - consumption-based модель, плата за единицы аналитики (AU), потребленные заданием

Аналог: AWS Glue
 
 - **Azure Data Lake Storage** - масштабируемое хранилище больших данных

Классификация: **PaaS/SaaS**, т.к. предоставляется платформа для хранения и обработки данных, но можно рассмотреть и как готовое хранилище

Тип оплаты: **Data Storage, Transactions** - оплата за объем хранимых данных и/или за транзакции

Аналог:  Amazon S3
 
 - **Azure Data Lake** - экосистема сервисов платформы Azure Data Lake (Data Transfer, Management и т.д.)

Классификация: **PaaS**, т.к. управляемая платформа для работы с большими данными

Тип оплаты: **Geo Replicated, Geo Redundant, Data Management, Data Services** - плата за дополнительные сервисы обработки данных (репликацию, избыточность и т.д.)

Аналог:  AWS Lake Formation
 
 - **Dynamics 365 Customer Insights** - AI-платформа для анализа данных клиентов и построения единого профиля клиента 360°

Классификация: **SaaS**, т.к. готовое CRM-решение с искусственным интеллектом для аналитики клиентских данных, не требующее управления инфраструктурой или разработки приложений

Тип оплаты: **Daily Unit** - оплата за ежедневное использование сервиса, зависящая от объёма обрабатываемых данных и количества анализируемых клиентских профилей

Аналог: Amazon Connect Customer Profiles
 
 - **Azure Operational Insights** - управляемая служба для сбора, анализа, мониторинга и визуализации операционных данных, телеметрии и логов с инфраструктуры, приложений и сетей Azure.

Классификация: **PaaS**, т.к. предоставляет платформу для аналитики операционных данных, но не готовое решение "из коробки" — требует конфигурации под конкретные нужды мониторинга

Тип оплаты: **Health Check** операции (в контексте интеграции с Network Watcher) — оплата за сетевые проверки здоровья
Данный сервис связан практически со всеми остальными, так как выступает центральным узлом для сбора и анализа данных в экосистеме Azure.

Аналог: Amazon CloudWatch Logs

4) **Artificial Intelligence** - платформы и инструменты для машинного обучения и AI

 - **Azure Machine Learning** - платформа для создания, обучения и развертывания ML-моделей

Классификация: **PaaS**, т.к. предоставляются инструменты и управляемая среда для работы с ML, а не готовая модель

Аналог:  Amazon SageMaker
 
 - **Azure AI Search** - служба поиска с искусственным интеллектом для создания мощных поисковых решений по корпоративным данным

Классификация: **PaaS**, т.к. предоставляет платформу для создания собственных поисковых решений (настройка индексов, схем, релевантности), а не готовый поисковик

Тип оплаты: **Bandwidth** - оплата за исходящий трафик (экспорт обученной модели, передача данных кленту)

Аналог: Amazon Kendra

### Database (Аналог в AWS: Storage)

Назначение: сервисы для хранения и передачи структурированных и неструктурированных данных.

 - **Azure Data Box** - физическое устройство для безопасной массовой оффлайн-передачи больших объёмов данных в облако Azure

Классификация: **IaaS**, т.к. предоставляет физическое железо (дисковые массивы) для передачи данных, пользователь управляет загрузкой данных на устройство и их транспортировкой

Тип оплаты: **Database Transfer** - оплата за объём переданных данных

Аналог: AWS Snowball
 
 - **Azure SQL** - масштабируемое облачное хранилище данных для аналитики и бизнес-интеллекта

Классификация: **PaaS**, т.к. полностью управляемая служба хранилища данных, пользователь работает только с данными и запросами

Тип оплаты: **Compute Optimized** - оплата за резервированные единицы мощности хранилища данных, масштабируемые в зависимости от вычислительных потребностей

Аналог: Amazon Redshift / Amazon RDS

### Network (Аналог в AWS: Network)

Назначение: сервисы для построения, защиты, управления и мониторинга сетевой инфраструктуры в облаке Azure.

 - **Azure Application Gateway** - балансировщик нагрузки уровня приложений с возможностями Web Application Firewall

Классификация: PaaS (Managed Service), т.к. полностью управляемый сервис балансировки нагрузки, пользователь настраивает только правила маршрутизации, SSL-сертификаты и политики

Тип оплаты: **Fixed Cost** - фиксированная плата за развёртывание и базовую доступность шлюза

**Capacity Units** - оплата за единицы пропускной способности (мощность обработки запросов)

**Data, Gateway %** - дополнительные метрики для Basic/WAF версий (оплата за объём данных или операции шлюза)
 
 - **Azure Network Watcher** - инструменты для мониторинга, диагностики и метрик сетевых ресурсов Azure. Данный сервис представлен в двух категориях (Cloud Services/Analytics и Networking), что демонстрирует его кроссфункциональность и взаимную интегрированность систем Azure.

Классификация: **PaaS**, т.к. предоставляет набор инструментов и API для анализа сети (мониторинг соединений, диагностика VPN, проверка потоков IP), но не готовое решение "из коробки", требует конфигурации под конкретные нужды

Тип оплаты: **Diagnostic API Tool** - оплата за использование диагностических API (в категории Cloud Services/Analytics)

**Network Watcher** - оплата за операции Health Check и мониторинга (в категории Networking)

Аналог: AWS Network Services

## Выводы
1) Большинство сервисов Azure - PaaS, что соответствует позиционируемой модели "платформа как сервис". AWS предоставляет больший баланс между IaaS/PaaS сервисами и соответственно больший контроль.
2) Оплата в основном за ресурсы, а не за инфраструктуру - либо за выделенные мощности (capacity-based), либо за конкретные операции/используемый объём хранилища (consumption-based). Реже встречается оплата за время использования (duration-based для IaaS, subscription-based для SaaS).
3) Сервисы интегрированы друг с другом, представляя единую экосистему, и в зависимости от выполняемых операций могут фигурировать в разных категориях. AWS напротив более модульный, предоставляет различные сервисы для разных задач.
